# 左側ファインダー 🚗⬅️

運転中、進行方向の**左側にあるコンビニ・ガソリンスタンドだけ**を地図に表示する運転支援Webアプリ。

> 右折で入りづらい店を避け、「左側にある＝スッと入れる」店だけを見つけるためのアプリ。

## 公開URL（スマホのブラウザで開く）

- **メイン**: https://atdirakg9020121-afk.github.io/left-finder/
- 旧/併存: https://atdirakg9020121-afk.github.io/left-side-finder/

HTTPS公開済みなので、位置情報・方位センサーが利用できます。

## 使い方

1. スマホのブラウザ（Safari / Chrome）で上のURLを開く
2. **「スタート」**をタップ → 位置情報を許可（iPhoneは方位センサーの許可も）
3. 地図に現在地が表示され、**進行方向の左側の施設が大きいピン＋店名**で出る
4. 走ると地図ごと回転し、常に「進行方向＝画面の上」「左側＝画面の左」になる

## 主な機能

| 機能 | 説明 |
|------|------|
| 左右判定 | 進行方向に対する施設の相対角から左右を計算（画面ではなく**進行方向基準**） |
| ヘディングアップ | 地図ごと回転し、進行方向が常に画面の上（カーナビ式）。自車は画面の下寄り（手前）に配置。回転は毎フレーム補間で滑らかに追従 |
| 次の左側施設 | ステータスに、進行方向で最も近い左前方の施設名＋距離を表示（左右が向きに追従して切替） |
| 自車マーカー | 青い丸＋進行方向を指す大きな三角。ズームしても位置がずれない |
| 店名ラベル | 左側施設に短縮店名＋距離を表示。地図回転を打ち消して常に水平で読める |
| ブランド色分け | セブン=オレンジ / ローソン=青 / ファミマ=緑 / GS=赤 / その他=紫 |
| 営業判定 | OpenStreetMapの営業時間から「24h / 営業中 / 営業時間外」を表示（確実に判定できる場合のみ） |
| 距離フィルタ | 200m〜1km で表示範囲を絞れる（既定500m） |
| 音声案内 | 「左、200メートル、ファミマ」と読み上げ。左前方250m以内を1回だけ案内（トグルON/OFF、既定OFF） |

## 技術構成

- **単一HTMLファイル**（`index.html`）で完結。ビルド不要
- 地図: [Leaflet](https://leafletjs.com/) + OpenStreetMapタイル
- 施設データ: [OpenStreetMap Overpass API](https://overpass-api.de/)（無料）
- 進行方向: 走行中（>2m/s）はGPSのheading、低速/停車時はコンパス（DeviceOrientation）に自動切替
- 音声: ブラウザ内蔵の Web Speech API（`speechSynthesis`）
- ホスティング: GitHub Pages（静的サイト、サーバーなし）

## 開発の経緯

「運転中、コンビニやガソスタを探すとき、右側にあると入りづらい。GPSで進行方向の左側にある施設だけ表示できないか」という発想からスタート。**まず動くものを作り、実際にスマホで試しながら仕様を詰める**進め方で開発した。

### 経緯ログ（2026-06-28）

1. **初版** — 左側施設のみを地図にピン表示。左右判定ロジックを全方位でテスト（10/10 PASS）し、Overpass APIで実データ取得・地図描画まで確認。
2. **位置情報取得の改善** — スマホで「取得失敗」が発生。原因は①ブロック②屋内でのタイムアウト。低精度で先行取得→高精度監視へ切替、タイムアウト延長、エラー理由を日本語表示、ブロック検知を追加。
3. **ヘディングアップ・マーカー固定・進行方向矢印** — 「画面に対して左右に見える」「ズームで丸が変な動き」「進行方向を向きたい」という指摘に対応。地図ごと回転するカーナビ式へ。自車マーカーをdivIcon化してズーム固定。矢印は地図回転を打ち消して常に画面の上を指す。
4. **店名ラベル** — 「丸だけだと何の店か分からない」に対応。左側施設に短縮店名＋距離を表示。地図回転を逆回転で打ち消して常に水平で読めるように。
5. **音声案内** — 地図アプリの重要機能を調査し、運転中に最も安全な音声案内を実装。左前方250m以内の最も近い施設を1回だけ読み上げ。連続発話の抑制・再接近での再案内・iOS音声解禁に対応。
6. **営業判定・距離フィルタ・ブランド色分け・矢印強調** — 営業時間判定（24h/営業中/時間外）、距離フィルタ、チェーン別の色分け、自車を画面下寄りに＋矢印を大きく。色はチェーンに使い、左右は大きさ・濃さ・ラベルの有無で区別する設計に。
7. **スワイプ不具合と重さの修正** — 「北以外でスワイプが変」「重い」に対応。手動ドラッグを無効化（GPS自動追従のみ）して回転スワイプ問題を根本解消。描画をrequestAnimationFrameで集約し、施設ピンの再生成を「向き3°以上変化 or 25m移動 or フィルタ変更 or 新データ」時のみに抑制して軽量化。

### 経緯ログ（2026-06-29）

8. **回転の滑らか化・矢印を手前・次の左側施設名** — 「地図回転がガクガクする」「矢印もっと手前でいい」「上の表示を施設名に」に対応。
   - **回転の滑らか化**: カクつきの原因は、方位が来るたびCSS `transition` を毎回リスタートして補間が途中で切れていたこと。CSS transitionを廃し、方位入力は目標角だけ更新→毎フレーム動く補間ループ（`rotationTick`、目標へ`ease`で寄せる）方式に変更。飛び飛びのコンパス入力でも描画が連続しカクつかない。359°↔0°の境界も最短経路で回る。
   - **自車を手前へ**: `#mapwrap` の `top` を 66%→78%→**85%** に下げ、自車（矢印）を画面のより手前に。前方を広く表示。
   - **次の左側施設**: ステータス「左側の施設（件数）」を**「次の左側施設」**に変更し、進行方向で最も近い左前方（±100°以内）の施設名＋距離を表示。該当なしは「前方になし／なし」、方向不明は「--」。

### 開発方針

- 変更のたびに**実際にコードを動かして検証**（左右判定はNode、UIはPlaywrightで擬似走行・スクショ確認）
- 正常系・異常系（0件・屋内・ブロック・レート制限など）・境界を確認
- 見つけた不具合は直してから再検証し、見つけた事実も正直に記録

## 既知の課題・今後の候補

- ラベルの密集を間引く（都心部で重なりやすい）
- オフライン耐性（取得済み施設をキャッシュし、圏外・トンネルでも消えないように）
- ガソリン価格表示（OSMに価格が無く、gogo.gs等のAPIはキー必須＋CORS制限。静的サイトでは中継サーバーが必要）
- コンパスの方位精度の改善
- 自車位置の下寄せ量（現状85%）・回転補間の機敏さ（`ROT_EASE=0.18`）の実機調整
