# 左側ファインダー 🚗⬅️

運転中、進行方向の**左側にあるコンビニ・ガソリンスタンドだけ**を地図に表示する運転支援Webアプリ。

> 右折で入りづらい店を避け、「左側にある＝スッと入れる」店だけを見つけるためのアプリ。

## 公開URL（スマホのブラウザで開く）

- **メイン**: https://atdirakg9020121-afk.github.io/left-finder/
- 旧/併存: https://atdirakg9020121-afk.github.io/left-side-finder/

HTTPS公開済みなので、位置情報・方位センサーが利用できます。

## 使い方

1. スマホのブラウザ（Safari / Chrome）で上のURLを開く
2. **「スタート」**をタップ → 位置情報を許可（iPhoneは方位センサーの許可も）
3. 地図に現在地が表示され、**進行方向の左側の施設が大きいピン＋店名**で出る
4. 走ると地図ごと回転し、常に「進行方向＝画面の上」「左側＝画面の左」になる

## 主な機能

| 機能 | 説明 |
|------|------|
| 左右判定 | 進行方向に対する施設の相対角から左右を計算（画面ではなく**進行方向基準**） |
| ヘディングアップ | 地図ごと回転し、進行方向が常に画面の上（カーナビ式）。自車は画面の下寄り（手前）に配置。回転は毎フレーム補間で滑らかに追従 |
| 次の左側施設 | ステータスに、進行方向で最も近い左前方の施設名＋距離を表示（左右が向きに追従して切替） |
| 自車マーカー | 青い丸＋進行方向を指す大きな三角。ズームしても位置がずれない |
| 店名ラベル | 左側施設に短縮店名＋距離を表示。地図回転を打ち消して常に水平で読める |
| ブランド色分け | セブン=オレンジ / ローソン=青 / ファミマ=緑 / GS=赤 / その他=紫 |
| 営業判定 | OpenStreetMapの営業時間から「24h / 営業中 / 営業時間外」を表示（確実に判定できる場合のみ） |
| 距離フィルタ | 500m / 1km / 2km / 3km / 5km で表示範囲を絞れる（既定500m、車での利用を想定したレンジ） |
| ブランドフィルタ | 下部の凡例（セブン/ローソン/ファミマ/GS/その他）をタップで表示ON/OFF。OFFは半透明＋取り消し線。地図・音声案内の両方に連動。都心など施設が多く重い場所で表示対象を絞れる |
| 音声案内 | 「左、200メートル、ファミマ」と読み上げ。左前方250m以内を1回だけ案内（トグルON/OFF、既定OFF） |

## 技術構成

- **単一HTMLファイル**（`index.html`）で完結。ビルド不要
- 地図: [Leaflet](https://leafletjs.com/) + OpenStreetMapタイル
- 施設データ: [OpenStreetMap Overpass API](https://overpass-api.de/)（無料）
- 進行方向: 走行中（>2m/s）はGPSのheading、低速/停車時はコンパス（DeviceOrientation）に自動切替
- 音声: ブラウザ内蔵の Web Speech API（`speechSynthesis`）
- ホスティング: GitHub Pages（静的サイト、サーバーなし）

## 開発の経緯

「運転中、コンビニやガソスタを探すとき、右側にあると入りづらい。GPSで進行方向の左側にある施設だけ表示できないか」という発想からスタート。**まず動くものを作り、実際にスマホで試しながら仕様を詰める**進め方で開発した。

### 経緯ログ（2026-06-28）

1. **初版** — 左側施設のみを地図にピン表示。左右判定ロジックを全方位でテスト（10/10 PASS）し、Overpass APIで実データ取得・地図描画まで確認。
2. **位置情報取得の改善** — スマホで「取得失敗」が発生。原因は①ブロック②屋内でのタイムアウト。低精度で先行取得→高精度監視へ切替、タイムアウト延長、エラー理由を日本語表示、ブロック検知を追加。
3. **ヘディングアップ・マーカー固定・進行方向矢印** — 「画面に対して左右に見える」「ズームで丸が変な動き」「進行方向を向きたい」という指摘に対応。地図ごと回転するカーナビ式へ。自車マーカーをdivIcon化してズーム固定。矢印は地図回転を打ち消して常に画面の上を指す。
4. **店名ラベル** — 「丸だけだと何の店か分からない」に対応。左側施設に短縮店名＋距離を表示。地図回転を逆回転で打ち消して常に水平で読めるように。
5. **音声案内** — 地図アプリの重要機能を調査し、運転中に最も安全な音声案内を実装。左前方250m以内の最も近い施設を1回だけ読み上げ。連続発話の抑制・再接近での再案内・iOS音声解禁に対応。
6. **営業判定・距離フィルタ・ブランド色分け・矢印強調** — 営業時間判定（24h/営業中/時間外）、距離フィルタ、チェーン別の色分け、自車を画面下寄りに＋矢印を大きく。色はチェーンに使い、左右は大きさ・濃さ・ラベルの有無で区別する設計に。
7. **スワイプ不具合と重さの修正** — 「北以外でスワイプが変」「重い」に対応。手動ドラッグを無効化（GPS自動追従のみ）して回転スワイプ問題を根本解消。描画をrequestAnimationFrameで集約し、施設ピンの再生成を「向き3°以上変化 or 25m移動 or フィルタ変更 or 新データ」時のみに抑制して軽量化。

### 経緯ログ（2026-06-29）

8. **回転の滑らか化・矢印を手前・次の左側施設名** — 「地図回転がガクガクする」「矢印もっと手前でいい」「上の表示を施設名に」に対応。
   - **回転の滑らか化**: カクつきの原因は、方位が来るたびCSS `transition` を毎回リスタートして補間が途中で切れていたこと。CSS transitionを廃し、方位入力は目標角だけ更新→毎フレーム動く補間ループ（`rotationTick`、目標へ`ease`で寄せる）方式に変更。飛び飛びのコンパス入力でも描画が連続しカクつかない。359°↔0°の境界も最短経路で回る。
   - **自車を手前へ**: `#mapwrap` の `top` を 66%→78%→**85%** に下げ、自車（矢印）を画面のより手前に。前方を広く表示。
   - **次の左側施設**: ステータス「左側の施設（件数）」を**「次の左側施設」**に変更し、進行方向で最も近い左前方（±100°以内）の施設名＋距離を表示。該当なしは「前方になし／なし」、方向不明は「--」。

9. **距離フィルタを車向けレンジに変更** — 「車で使うから」という用途に合わせ、距離フィルタの選択肢を 200m/300m/500m/800m/1km から **500m / 1km / 2km / 3km / 5km** に変更（既定は500mのまま）。あわせて施設取得半径 `FETCH_RADIUS` を 1km→**5km** に拡大（取得半径が1kmのままだと2km以上を選んでもフィルタが効かないため、最大値に合わせる必須の連動修正）。Playwrightでセレクト内容とフィルタ整合を確認。

10. **ブランドフィルタ（凡例タップ）** — 取得半径を5kmに広げた結果、都心部で施設が増えて重くなったため、表示対象を絞れるように。下部の凡例（セブン/ローソン/ファミマ/GS/**その他**）を**タップで表示ON/OFF**できるようにした。`brandColor` と同基準の分類関数 `brandKey()` を新設し、`activeBrands`（Set）で管理。描画ループで非表示ブランドのマーカー生成自体をスキップして軽量化。音声案内も非表示ブランドは読み上げないよう連動。OFFは凡例を半透明＋取り消し線で表示。Playwrightで分類・マーカー増減（全ON5→絞り込み→全OFF0→復元5の往復）・実タップ・見た目を検証。

11. **今いる道路だけ表示（道ごとの絞り込み）** — 「他の道の店まで出るとごちゃごちゃ」に対応。左右判定は進行方向ベースのままだったため、別の道沿いの店も全部「左側」として拾っていた。Overpassで周辺の道路(way)も取得し、自分が乗っている道路を特定→その道路ラインから一定距離内の店だけ表示する方式に。下部に「今の道だけ」チェックボックスを追加（OFFで従来の全件表示／道路特定不能時は自動で全件にフォールバック）。音声案内も同じ道沿いに連動。
    - **OSMの道路分断対策**: 同じ通りが多数のwayに割れているため、`name` が一致するway／番号道路は `ref`（国道番号など）が一致するwayを束ねて1本の道路として扱う。名前も番号も無い道は、向きが平行で連続する近接wayを束ねる。
    - **都心・郊外の両対応（重要）**: 検証で都心と郊外の差が判明。郊外の国道は区間ごとに `name` がバラつき（○○バイパス/△△街道/無名）、道路中心線から店までの距離も都心の裏道(24m)に対し郊外バイパスは46m以上と大きく違う。一律45m閾値では郊外で道沿いの店を取りこぼしたため、**道路種別(highway tag)ごとに沿道判定の閾値を変える**ようにした（幹線/バイパス=75m、生活道路=40m など）。これで都心は隣の道を巻き込まず、郊外バイパスは道沿いの店を拾える。
    - Playwrightで都心（東京駅）と郊外（宇都宮国道119号・栃木矢板・茨城国道50号）の実データを取得し、道路特定・束ね・絞り込み・ON/OFF往復・異常系を両エリアで検証。

12. **収益化の検討と「寝かせ」判断（OGPのみ残置）** — 「お金を稼ぐレベルにしたい」という相談から収益化を検討。節約ドライバー向けには**ガソリン価格×左側送客**が本命だが、ガソリン価格APIはキー必須＋CORS制限で静的サイトから直叩き不可、かつ集客（アクセスほぼ0）が最大のボトルネックという結論に。一度 OGP/Twitterカード・アクセス解析（Cloudflare Web Analytics）・アフィリ枠「⛽お得に給油」を `index.html` に実装しPlaywrightで検証まで通したが、**「発案直後でまずアプリの仕様を詰めるのが先」**という方針に基づき、**SNS共有の土台になるOGPだけ残し、解析タグとアフィリ枠は撤去**した。収益化プランの詳細は寝かせて並行管理（実装は仕様確定後に再開）。

13. **ラベル間引き・オフライン耐性** — アプリの完成度を上げる改善（※この時点でまだ一度も実車検証していない＝以下はPlaywrightの擬似走行で検証）。
    - **ラベル間引き（都心の重なり対策）**: 左側施設のラベル（店名＋距離）が都心で重なって読めない問題に対応。描画対象を一度集めて**距離順にソート**し、ラベルを置くたびに画面座標(`latLngToContainerPoint`)を記録、既存ラベルと画面距離が `LABEL_MIN_GAP_PX`(=46px)未満なら後勝ち(=より遠い店)のラベルを省いてドットだけにする。**近い店ほどラベルを残す**（運転中は手前の店が重要）。Playwrightで密集5→ラベル1・離散→ほぼ全部・郊外低密度→間引かない、を確認。`LABEL_MIN_GAP_PX` は実車で最適値を詰めるため定数化。
    - **オフライン耐性（トンネル・圏外）**: 施設・道路を `localStorage`(`lsf_cache_v1`)にキャッシュ。取得成功時に保存し、①起動直後に先読み（圏外起動でも白画面にしない）②取得失敗かつ施設が空のときキャッシュ復元して「オフライン表示中」を案内。電波が戻れば自動で再取得・上書き。Playwrightで成功→保存、失敗→復元、キャッシュ無し失敗→従来メッセージ（壊れない）を検証。

### 開発方針

- 変更のたびに**実際にコードを動かして検証**（左右判定はNode、UIはPlaywrightで擬似走行・スクショ確認）
- **位置・地図に関わる機能は必ず都心と郊外の両方の実データで検証する**。OpenStreetMapは施設密度・道路のname/ref・道幅が都心と郊外で根本的に違い、都心だけで「動いた」とすると郊外で破綻する（実例：今いる道だけ表示の閾値）。距離・密度の閾値は道路種別ごとに変える発想を持つ。
- 正常系・異常系（0件・屋内・ブロック・レート制限など）・境界を確認
- 見つけた不具合は直してから再検証し、見つけた事実も正直に記録

- ~~オフライン耐性（取得済み施設をキャッシュし、圏外・トンネルでも消えないように）~~ → 経緯13で実装
- ガソリン価格表示（OSMに価格が無く、gogo.gs等のAPIはキー必須＋CORS制限。静的サイトでは中継サーバーが必要）
- **コンパスの方位精度の改善**（※実機センサーでしか善し悪しを評価できない＝実車検証待ち）
- **自車位置の下寄せ量（現状85%）・回転補間の機敏さ（`ROT_EASE=0.18`）の実機調整**（※走ってみないと正解が決まらない＝実車検証待ち。実車でスライダー調整できるUIを付ける案あり）

> ⚠️ **まだ一度も実車検証していない**（発案から数日・擬似走行のみ）。位置・地図・センサー系は実車で挙動が変わるため、コンパス精度と回転補間・自車下寄せ量は実走後に最終調整する。
